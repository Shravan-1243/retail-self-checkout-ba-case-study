**Analyst:** Shravan Kanamadi  
**Type:** Business Analysis Portfolio Project  
**Status:** Complete

---

## Project Overview

Self-checkout shrinkage is one of the fastest-growing sources of 
retail loss in the UK. This case study applies end-to-end business 
analysis methodology to identify the root causes of self-checkout 
process failure at Greenfield Stores and define the requirements 
for an improvement solution.

The project uses real published data sources (ECR Community, 
British Retail Consortium) for market context, with clearly labelled 
illustrative data for store-level analysis.

---

## Business Problem

Greenfield Stores is experiencing a 2.3% shrinkage rate at 
self-checkout terminals - above the UK retail average. Losses 
are driven by a combination of process gaps, system limitations, 
and inconsistent staff intervention. Leadership wants to understand 
the problem and define requirements for a solution before committing 
to a technology investment.

| Stakeholder Register | Power/interest grid (Mendelow), RACI matrix, engagement approach |
| Illustrative Dataset | Store-level transaction and shrinkage data with data dictionary |
| Interactive Dashboard | Visual summary of shrinkage patterns and KPIs |

## Methodology

- Requirements elicitation via stakeholder analysis
- AS-IS process mapping to identify failure points
- Gap analysis against industry benchmarks
- TO-BE process design aligned to business requirements
- BPMN notation throughout
